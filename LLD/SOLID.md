# **S.O.L.I.D Principles**

These are the design principles to be followed to write scalable, maintainable code

 1. S - *Single Responsibility Principle **[SRP]**.*
 2. O - *Open/Close Principle **[OCP]**.*
 3. L - *Liskov Dependance Principle **[LDP]**.*
 4. I - *Inteface Segregation Principle **[ISP]**.*
 5. D - *Dependance Inversion Principle **[DIP]**.*

 ## **Single Responsibility Principle [SRP]**

 Each class should have only one reason to change.
